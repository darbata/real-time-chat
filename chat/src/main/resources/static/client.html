Connection
Setup reminder
  1. Spring Boot: register a WebSocketMessageBrokerConfigurer with registerStompEndpoints("/ws") and enable a simple broker on /topic, /queue.
  2. SockJS — use http:// URL (it negotiates transport). Native WS — use ws:// URL.
  3. Add auth headers here (e.g. Authorization: Bearer …) — they're sent in the STOMP CONNECT frame.
Subscriptions
Destination patterns
  • /topic/… — broadcast to all subscribers (pub/sub)
  • /queue/… — point-to-point (one consumer)
  • /user/queue/… — user-specific private queue
Subscribe first, then send to /app/… — the server's @MessageMapping handler will route the response back to the topic/queue you're subscribed to via @SendTo.
    Publish
    Sending messages
    1. Send to /app/… — this hits your @MessageMapping controller methods. The prefix (/app) is set by setApplicationDestinationPrefixes().
    2. The controller return value + @SendTo("/topic/…") determines where the response goes.
    3. For JSON payloads, set content-type to application/json — Spring will auto-deserialize to your DTO via Jackson.
    4. You can also send directly to /topic/… to bypass the controller (goes straight to broker).

    Quick Reference
    1. Connect — point to your STOMP endpoint (e.g. http://localhost:8080/ws)
    2. Subscribe — listen on a destination like /topic/messages
    3. Publish — send to /app/hello (hits @MessageMapping) or directly to /topic/… (bypasses controller)
    4. Watch the log — filter by type (Sent/Received/Errors) or search text
